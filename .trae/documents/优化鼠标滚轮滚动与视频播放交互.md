# 优化鼠标滚轮滚动与视频播放交互方案

## 问题分析
当前视频播放断断续续的主要原因是：
1. 快速滚动时，`syncVideoToScroll` 被频繁调用，导致视频 `currentTime` 被频繁更新
2. 没有根据滚动状态动态调整视频播放策略
3. 缺乏滚动停止检测机制

## 优化方案

### 1. 添加滚动状态检测机制
- 添加 `isScrollingRef` 标记用户是否正在滚动
- 添加 `scrollStopTimeoutRef` 用于检测滚动停止
- 添加 `scrollVelocityThreshold` 定义快速滚动阈值

### 2. 优化滚动处理函数
- 在 `handleScroll` 中添加滚动状态检测
- 设置滚动停止定时器（100ms）
- 根据滚动速度调整视频更新策略

### 3. 优化视频时间更新逻辑
- 在快速滚动时，减少视频时间更新频率
- 仅在滚动速度较低或滚动停止时，进行精确的视频时间更新
- 实现分级更新策略：
  - 快速滚动：每50ms更新一次视频时间
  - 慢速滚动：每16ms更新一次视频时间
  - 滚动停止：立即更新到精确位置

### 4. 优化鼠标滚轮事件处理
- 在 `handleWheel` 中标记滚动状态
- 触发滚动停止检测

### 5. 实现滚动停止恢复逻辑
- 当检测到滚动停止时，立即将视频更新到精确位置
- 恢复正常的视频更新频率

## 预期效果
- 快速滚动时，视频播放流畅，无卡顿现象
- 滚动停止后，视频立即同步到正确位置
- 平衡滚动体验与视频播放流畅性
- 提高整体页面性能

## 实现步骤
1. 添加滚动状态相关的ref和常量定义
2. 修改 `handleScroll` 函数，添加滚动状态检测和定时器
3. 优化 `syncVideoToScroll` 函数，实现分级视频更新策略
4. 修改 `handleWheel` 函数，触发滚动状态更新
5. 添加滚动停止恢复逻辑
6. 测试优化效果，确保功能正常
