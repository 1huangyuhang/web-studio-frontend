## 问题分析
1. **当前组件的逻辑问题**：
   - 滚动进度计算方式不准确，基于页面滚动位置而非组件内部滚动
   - 组件定位方式（absolute）可能导致在不同页面布局下显示异常
   - 动画过渡不够流畅，缺乏层次感
   - 缺乏对各种滚动场景的适配

2. **dify 官网页脚动画特点**：
   - 页脚固定在页面底部，内容区域滚动时页脚保持不动
   - 滚动到页面底部时，内容区域平滑向上移动，逐渐揭示页脚
   - 动画过渡流畅，有明确的层次感和视觉引导
   - 响应式设计，在不同设备上都有良好表现

## 修改方案

1. **重新设计组件结构**：
   - 将组件改为使用相对定位，避免绝对定位带来的布局问题
   - 分离内容区域和页脚区域，确保页脚固定在页面底部
   - 优化组件的层级结构，增强视觉层次感

2. **优化滚动逻辑**：
   - 使用 Intersection Observer API 来检测组件在视口中的位置
   - 基于组件可见性计算滚动进度，而非页面滚动位置
   - 确保滚动进度计算准确，动画过渡流畅

3. **调整动画效果**：
   - 增加动画的层次感，使用多种过渡效果（透明度、缩放等）
   - 调整过渡时间和缓动函数，使其更加符合 dify 官网的风格
   - 增加视觉引导，提示用户可以滚动查看更多内容

4. **修复定位问题**：
   - 确保组件在各种页面布局下都能正常显示
   - 修复响应式设计中的布局问题
   - 确保页脚始终固定在页面底部
