## 问题分析
用户希望视频容器（.scrolly-video-container）的宽度能填充满左右两边的页面宽度，而不是被限制在内容区域内。

## 解决方案
1. **修改视频容器样式**：为 `.scrolly-video-container` 添加全屏宽度样式，突破父容器的宽度限制
2. **确保视频元素全屏**：确保视频元素本身也能全屏显示，不被限制
3. **移除宽度限制**：移除可能限制宽度的 `max-width` 设置

## 具体实现
### 修改 `/src/animations/ScrollyVideo.less`
```less
// 添加全屏宽度样式
.scrolly-video-container {
  position: relative;
  width: 100vw; // 确保宽度为视口宽度
  margin-left: -50vw; // 向左偏移50%视口宽度
  margin-right: -50vw; // 向右偏移50%视口宽度
  left: 50%; // 定位中心点
  right: 50%;
  max-width: none; // 移除最大宽度限制
  // 其他现有样式...
}

// 确保视频元素全屏显示
.scrolly-video-element {
  width: 100vw !important;
  height: 100vh !important;
  // 其他现有样式...
}
```

## 预期效果
- 视频容器宽度填充满左右两边的页面宽度
- 视频元素全屏显示，覆盖整个视口
- 后续内容依然在正常的内容区域内显示
- 不影响其他页面元素的布局

## 验证方法
1. 构建项目，确保没有编译错误
2. 在浏览器中查看效果，确认视频容器宽度填充满页面
3. 滚动页面，确认后续内容正常显示

这个方案借鉴了项目中已经使用的全屏宽度实现方式（如 `.action-section-container` 和 `.footer-section`），确保了代码的一致性和可靠性。
