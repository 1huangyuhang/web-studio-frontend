# 实现页面游览进度条

## 1. 需求分析

* 在页面顶部添加一个游览进度条，样式颜色为符合整个页面的主题

* 进度条随用户滚动实时更新，反映当前阅读位置

* 包含平滑的动画效果

* 固定在页面顶部，不影响其他内容

## 2. 实现方案

### 2.1 创建进度条组件

* 创建一个新的`ProgressBar`组件，位于`/src/components/ui/`目录下

* 组件包含一个固定定位的进度条元素

* 监听页面滚动事件，计算滚动进度

* 使用CSS过渡动画实现平滑的宽度变化

### 2.2 集成到布局组件

* 在`Layout`组件中引入并使用`ProgressBar`组件

* 确保进度条位于页面最顶层，z-index设置合理

* 与现有的导航栏和其他固定元素兼容

### 2.3 实现滚动监听

* 使用`useEffect`钩子添加和移除滚动事件监听

* 计算滚动进度：`(window.scrollY / (document.body.scrollHeight - window.innerHeight)) * 100`

* 使用`useState`保存进度值，触发组件重新渲染

### 2.4 添加动画效果

* 使用CSS过渡动画实现进度条宽度变化

* 添加淡入效果，当页面开始滚动时显示进度条

* 优化性能，使用`requestAnimationFrame`或防抖函数

## 4. 技术实现

### 4.1 ProgressBar.tsx

```typescript
import React, { useState, useEffect } from 'react';
import './ProgressBar.less';

const ProgressBar: React.FC = () => {
  const [progress, setProgress] = useState(0);

  const handleScroll = () => {
    // 计算滚动进度
    const scrollTop = window.scrollY;
    const docHeight = document.body.scrollHeight - window.innerHeight;
    const scrollPercent = (scrollTop / docHeight) * 100;
    setProgress(scrollPercent);
  };

  useEffect(() => {
    // 添加滚动事件监听
    window.addEventListener('scroll', handleScroll, { passive: true });
    
    // 初始调用一次，设置初始状态
    handleScroll();
    
    // 清理函数
    return () => {
      window.removeEventListener('scroll', handleScroll);
    };
  }, []);

  return (
    <div className="progress-bar-container">
      <div 
        className="progress-bar" 
        style={{ width: `${progress}%` }}
      />
    </div>
  );
};

export default ProgressBar;
```

## 5. 性能优化

* 使用`passive: true`优化滚动事件监听

* 考虑添加防抖函数，减少滚动事件触发频率

* 使用CSS过渡动画，避免JavaScript动画的性能问题

* 当进度为0或100时，考虑隐藏进度条

* 添加适当的阴影效果，增强视觉层次感

* 确保在不同浏览器中表现一致

## 7. 测试和验证

* 测试不同页面的滚动效果

* 检查与其他固定元素的兼容性

* 测试动画效果的流畅度

通过以上实现，我们可以在页面顶部添加一个随滚动实时更新的进度条，提升用户体验，让用户随时了解当前的阅读进度。
