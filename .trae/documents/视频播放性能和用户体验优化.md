## Issues Identified in ScrollyVideo.tsx

1. **Video Visibility Issue**: `isVideoVisible` is initially `false`, causing the video to be invisible until scroll position changes
2. **Video Playback Issue**: Video is not explicitly played, which can prevent setting `currentTime` in some browsers
3. **Buffering State Management**: Missing proper handling of video buffering state
4. **Event Listener Issues**: Improper cleanup of video buffering event listeners
5. **Code Redundancy**: Unused variables and complex logic in `syncVideoToScroll` function

## Solution Plan

1. **Fix Video Visibility**: Set `isVideoVisible` initial value to `true`
2. **Ensure Video Playback**: Add `play()` call in `handleVideoLoaded` with error handling
3. **Implement Buffering State**: Use `isBufferingRef` from `useVideoBuffer` hook
4. **Add Buffering Event Listeners**: Add `waiting` and `playing` event listeners with proper cleanup
5. **Simplify Sync Logic**: Streamline `syncVideoToScroll` function for better performance
6. **Clean Up Code**: Remove unused variables and simplify logic

## Implementation Steps

1. Update `isVideoVisible` initial state to `true`
2. Modify `handleVideoLoaded` to include `play()` call
3. Add `isBufferingRef` to `useVideoBuffer` hook usage
